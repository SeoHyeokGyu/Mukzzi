## 프로젝트 개요

### 문제 정의

매일 반복되는 메뉴 선택은 누구에게나 피로한 일상입니다. 기존 솔루션들은 이 문제를 충분히 해결하지 못합니다:

- **배달앱 추천**: 광고 기반 추천, 개인 영양 상태 미반영
- **식단 관리 앱**: 기록 위주, 메뉴 결정을 도와주지 않음
- **랜덤 메뉴 앱**: 단순 랜덤, 지속 사용할 동기 부재

### 솔루션

**Mukzzi**는 재미있는 메뉴 결정 + 식사 기록 + 캐릭터 성장을 하나로 연결한 게이미피케이션 식사 앱입니다:

- **다양한 메뉴 결정 방식**: 랜덤 룰렛, 상황별 필터, 선호도 기반 추천으로 메뉴 선택 자체를 즐거운 경험으로
- **캐릭터 성장 시스템**: 식사 기록이 곧 먹찌 육성 — 퀘스트 달성, 영양소 기반 파츠 조합 성장
- **영양 피드백**: 영양소 DB 연동으로 섭취 밸런스를 분석하고 다음 메뉴 추천
- **패널티 & 리텐션**: 미기록 시 먹찌 배고픔 연출로 자연스러운 재방문 유도

## 핵심 기능

### 1. 메뉴 결정

#### 랜덤 룰렛
- 먹찌가 대신 룰렛을 돌려주는 인터랙티브 애니메이션
- 사용자 즐겨찾기 및 최근 기록 기반 메뉴 풀 구성

#### 상황별 필터
- 날씨·기분 태그 기반 맞춤 추천
- 조건 조합으로 상황에 딱 맞는 메뉴 제안

#### 선호도 기반 추천
- 기록 빈도, 시간대 패턴, 날씨·기분 패턴, 명시적 선호(좋아요/싫어요) 4가지 신호 조합
- 최근 3일 내 먹은 메뉴 제외, 신호별 가중합으로 개인화 추천 목록 제공
- 신규 유저는 전체 메뉴 DB 기반 일반 추천으로 폴백

### 2. 식사 기록

#### 메뉴 검색 & 등록
- 메뉴명 검색으로 간편 등록
- USDA + 식약처 영양소 DB 자동 매핑
- 즐겨찾기 관리로 자주 먹는 메뉴 빠른 접근

#### 먹부림 캘린더
- 날짜별 먹찌 아이콘 + 식사 기록 한눈에 확인
- 월간 식사 패턴 시각화

### 3. 캐릭터 성장 (먹찌)

#### 퀘스트 시스템
- **일일 퀘스트**: 식사 기록, 삼시세끼, 균형 식단, 새 메뉴 도전 — 진행 현황 추적 및 업적 조건 누적
- **주간 퀘스트**: 7일 연속 기록, 주간 영양 밸런스, 다양한 식단 — 완료 시 배경/이펙트/동작 보상
- **업적 퀘스트**: 식사 누적·연속·다양성 조건 달성 시 진화, 모션, 배경, 이펙트, 칭호, 악세서리 해금
- 식사 등록 직후 퀘스트 진행 현황 실시간 반영 + 완료 시 보상 연출

#### 성장 시스템 (파츠 조합)
- 최근 7일 영양소 비율이 모든 파츠에 종합적으로 영향
- 체형, 근육, 피부색, 표정 (각 5단계) 조합으로 625가지 외형 + 악세서리 (퀘스트 보상으로 획득)
- 달성한 외형은 도감에 기록, 이전 외형으로 변경 가능 (가역적)

#### 패널티 시스템
- 미기록 일수에 따른 단계적 패널티로 자연스러운 재방문 유도

| 미기록 일수 | 상태 | 영향 |
|------------|------|------|
| 2일 | 허기짐 | 먹찌 표정 변화 (슬픈 표정) |
| 3일 | 배고픔 | 푸시 알림 + 홈 화면 배고픔 연출 |
| 5일 이상 | 기력 저하 | 경험치 획득량 50% 감소 |

- 퀘스트 진행도·외형은 **하락하지 않음** (이탈 방지)
- 식사 1회 기록 시 즉시 복구

#### 먹찌 도감
- 달성했던 먹찌 외형 컬렉션
- 수집 현황 및 달성 조건 표시

### 4. 소셜

#### 친구 검색 및 추가
- 닉네임 또는 고유 ID로 사용자 검색
- 검색 탭에 추천 사용자 노출 (비슷한 식습관, 비슷한 먹찌, 인기 사용자)
- 친구 요청 -> 수락 방식

#### 프로필 방문
- 친구의 먹찌 외형, 뱃지, 도감 구경

#### 차단/신고
- 부적절한 사용자 차단 및 신고 처리

### 5. 영양 피드백

#### 영양 밸런스 분석
- 하루/주간 영양소 섭취 비율 시각화
- 부족한 영양소 기반 다음 메뉴 추천 멘트 제공

## 기술 스택

### Frontend (iOS / Android / Web)
- **Framework**: Flutter 3.24, Dart 3.5
- **State Management**: Riverpod
- **Animation**: Rive (먹찌 성장 인터랙티브 연출)
- **Routing**: GoRouter
- **HTTP**: Dio
- **Responsive**: 모바일 / 데스크톱 반응형 레이아웃 (LayoutBuilder)

### Backend
- **Language**: Go 1.26
- **Framework**: Gin (경량 HTTP 라우터)
- **ORM**: GORM (PostgreSQL 연동)
- **Scheduler**: robfig/cron (패널티 시스템, 푸시 알림 배치)
- **Real-time**: gorilla/websocket — Tier 3 그룹 기능 대비

### Database
- **Main DB**: PostgreSQL 15+ (영양소·캐릭터 관계 데이터)
- **Cache**: Redis 7+ (음식 검색 캐싱, 세션 관리)
- **File Storage**: Oracle Object Storage (음식 사진, 캐릭터 이미지)

### AI / 외부 API
- **음식 사진 인식**: Google Vision API (Tier 2)
- **영수증 OCR**: Naver Clova OCR (Tier 2)
- **메뉴 추천 AI**: OpenAI GPT API (Tier 2)
- **영양소 DB**: 식약처 API + USDA

### 인증
- **토큰**: JWT (Access + Refresh)
- **소셜**: Kakao OAuth 2.0 / Google OAuth / Apple Sign In

### 인프라
- **Server**: Oracle Cloud (ARM Ampere A1)
- **Container**: Docker + Docker Compose
- **Push**: FCM (Firebase Cloud Messaging)
- **CI/CD**: GitHub Actions

> 상세: [docs/infrastructure.md](docs/infrastructure.md)


### 핵심 데이터 플로우

#### 식사 기록 → 캐릭터 성장 플로우
```
사용자 → 메뉴 결정 (룰렛/필터/선호도 추천)
       → 메뉴 확정
       → 식사 기록 (메뉴명 검색 + 영양소 자동 매핑)
       → 기록 저장
       → 퀘스트 진행 현황 갱신 (일일/주간/업적)
       → (퀘스트 완료 시) 보상 지급 (배경/이펙트/동작/악세서리)
       → 영양소 비율 재계산 (최근 7일)
       → 파츠 조합 재계산 (체형, 근육, 피부색, 표정 각 5단계)
       → (외형 변화 시) 먹찌 성장 연출 + 도감 등록
```

#### 패널티 플로우
```
Cron Scheduler (매일 실행)
       → 유저별 마지막 기록일 체크
       → 미기록 일수 산정
       → 2일: 허기짐 (슬픈 표정)
       → 3일: 배고픔 (FCM 알림 + 홈 화면 연출)
       → 5일: 기력 저하 (경험치 획득량 50% 감소)
       → 기록 재개 시 즉시 복구
```

### 백엔드 아키텍처 (Layered Clean Architecture)

```
┌─────────────────────────────────────────────────┐
│           Delivery Layer (Gin HTTP/WS)          │
│  handler/ : Auth, User, Menu, MealRecord,       │
│             Character, Calendar, Nutrition,      │
│             Roulette, Friend                     │
│  dto/     : Request/Response DTO                │
└─────────────────────────────────────────────────┘
                      ↓ DTO
┌─────────────────────────────────────────────────┐
│              Usecase Layer                      │
│  비즈니스 로직, 영양소 계산, 파츠 조합 판정,       │
│  퀘스트 관리, 친구 검색/추천/차단 관리              │
└─────────────────────────────────────────────────┘
                      ↓ Domain Model
┌─────────────────────────────────────────────────┐
│              Domain Layer                       │
│  엔티티 구조체, 리포지토리 인터페이스, 도메인 에러   │
└─────────────────────────────────────────────────┘
                      ↑ implements
┌─────────────────────────────────────────────────┐
│            Repository Layer (GORM)              │
└─────────────────────────────────────────────────┘
                      ↓
┌──────────────────────────┬──────────────────────┐
│     PostgreSQL 15+       │      Redis 7+        │
│   (주 데이터베이스)        │   (캐싱 + 세션)      │
└──────────────────────────┴──────────────────────┘
```

## 개발 로드맵

### Tier 1 — MVP (출시 목표)
> 룰렛·상황별 필터로 메뉴를 고르고, 텍스트로 식사를 기록하면 먹찌가 성장합니다.

- 메뉴 결정 (룰렛, 상황별 필터, 선호도 기반 추천)
- 식사 기록 (텍스트 입력 + 영양소 DB 연동)
- 캐릭터 성장 (퀘스트 시스템, 파츠 조합 625가지, 패널티)
- 먹부림 캘린더 + 먹찌 도감
- 영양 밸런스 피드백
- 친구 시스템 (검색/추천 사용자/프로필 방문/차단/신고)
- 회원가입 / 로그인 (JWT + 소셜)

### Tier 2 — 고도화
> AI 사진 인식, 영수증 OCR, AI 메뉴 추천, 스트릭, 소셜 기능, 캐릭터 커스터마이징

### Tier 3 — 미래 확장
> 그룹 챌린지, 1:1 배틀, 위치 기반 퀘스트, 대화형 AI 상담

> 상세: [docs/planning.md](docs/planning.md)
