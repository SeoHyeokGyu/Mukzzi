## 🎯 프로젝트 개요

### 문제 정의

매일 반복되는 메뉴 선택은 누구에게나 피로한 일상입니다. 기존 솔루션들은 이 문제를 충분히 해결하지 못합니다:

- **배달앱 추천**: 광고 기반 추천, 개인 영양 상태 미반영
- **식단 관리 앱**: 기록 위주, 메뉴 결정을 도와주지 않음
- **랜덤 메뉴 앱**: 단순 랜덤, 지속 사용할 동기 부재

### 솔루션

**Mukzzi**는 재미있는 메뉴 결정 + 식사 기록 + 캐릭터 성장을 하나로 연결한 게이미피케이션 식사 앱입니다:

- **다양한 메뉴 결정 방식**: 랜덤 룰렛, 이상형 월드컵, 상황별 필터로 메뉴 선택 자체를 즐거운 경험으로
- **캐릭터 성장 시스템**: 식사 기록이 곧 먹찌 육성 — 경험치 획득, 레벨업, 영양소 기반 변신
- **영양 피드백**: 영양소 DB 연동으로 섭취 밸런스를 분석하고 다음 메뉴 추천
- **패널티 & 리텐션**: 미기록 시 먹찌 배고픔 연출로 자연스러운 재방문 유도

## ✨ 핵심 기능

### 1. 메뉴 결정

#### 랜덤 룰렛
- 먹찌가 대신 룰렛을 돌려주는 인터랙티브 애니메이션
- 사용자 즐겨찾기 및 최근 기록 기반 메뉴 풀 구성

#### 이상형 월드컵
- 두 메뉴를 비교하는 토너먼트 방식으로 최종 메뉴 선택
- 8강/16강 등 라운드 수 선택 가능

#### 상황별 필터
- 날씨·기분 태그 기반 맞춤 추천
- 조건 조합으로 상황에 딱 맞는 메뉴 제안

### 2. 식사 기록

#### 메뉴 검색 & 등록
- 메뉴명 검색으로 간편 등록
- USDA + 식약처 영양소 DB 자동 매핑
- 즐겨찾기 관리로 자주 먹는 메뉴 빠른 접근

#### 먹부림 캘린더
- 날짜별 먹찌 아이콘 + 식사 기록 한눈에 확인
- 월간 식사 패턴 시각화

### 3. 캐릭터 성장 (먹찌)

#### 경험치 & 레벨업
- **적립 규칙**: 식사 기록 30XP, 하루 3끼 완료 50XP, 신메뉴 20XP, 밸런스 달성 40XP
- **레벨 체계**: Lv.1~50, 구간별 필요 경험치 차등 (초반 빠른 성장 → 후반 도전)
- 식사 등록 직후 경험치 바 실시간 반영 + 레벨업 연출

#### 변신 시스템 (5종)
- 최근 7일 섭취 영양소 비율에 따라 먹찌 외형이 변신

| 변신 형태 | 조건 | 외형 컨셉 |
|-----------|------|-----------|
| 기본 먹찌 | 초기 상태 | 둥글고 순한 기본형 |
| 근육 먹찌 | 단백질 비율 40% 이상 | 단단하고 힘센 체형 |
| 채소 먹찌 | 식이섬유·비타민 비율 35% 이상 | 싱그럽고 건강한 외형 |
| 에너지 먹찌 | 탄수화물 비율 50% 이상 | 활발하고 통통한 외형 |
| 밸런스 먹찌 | 3대 영양소 편차 10% 이내 | 균형 잡힌 황금 외형 |

- 조건 이탈 시 이전 형태로 복귀 (도감에는 기록 유지)
- 변신 시 연출 애니메이션 재생 + 도감 자동 등록

#### 패널티 시스템
- 미기록 일수에 따른 단계적 패널티로 자연스러운 재방문 유도

| 미기록 일수 | 상태 | 영향 |
|------------|------|------|
| 2일 | 허기짐 | 먹찌 표정 변화 (슬픈 표정) |
| 3일 | 배고픔 | 푸시 알림 + 홈 화면 배고픔 연출 |
| 5일 이상 | 기력 저하 | 경험치 획득량 50% 감소 |

- 레벨·변신 형태는 **하락하지 않음** (이탈 방지)
- 식사 1회 기록 시 즉시 복구

#### 먹찌 도감
- 변신했던 먹찌 외형 컬렉션
- 수집 현황 및 달성 조건 표시

### 4. 영양 피드백

#### 영양 밸런스 분석
- 하루/주간 영양소 섭취 비율 시각화
- 부족한 영양소 기반 다음 메뉴 추천 멘트 제공

## 🛠️ 기술 스택

### Mobile (Frontend)
- **Framework**: React Native (iOS/Android 동시 개발)
- **State Management**: Zustand
- **Animation**: Reanimated 3 (먹찌 변신 연출)

### Backend
- **Core**: Spring Boot 3.2, Kotlin
- **Scheduler**: Spring Scheduler (패널티 시스템, 푸시 알림 배치)
- **Real-time**: WebSocket (STOMP) — Tier 3 그룹 기능 대비

### Database
- **Main DB**: PostgreSQL 15+ (영양소·캐릭터 관계 데이터)
- **Cache**: Redis 7+ (음식 검색 캐싱, 세션 관리)
- **File Storage**: AWS S3 (음식 사진, 캐릭터 이미지)

### AI / 외부 API
- **음식 사진 인식**: Google Vision API (Tier 2)
- **영수증 OCR**: Naver Clova OCR (Tier 2)
- **메뉴 추천 AI**: OpenAI GPT API (Tier 2)
- **영양소 DB**: 식약처 API + USDA

### 인증
- **토큰**: JWT (Access + Refresh)
- **소셜**: Kakao OAuth 2.0 / Google OAuth / Apple Sign In

### 인프라
- **Server**: AWS EC2 + RDS
- **Push**: FCM (Firebase Cloud Messaging)
- **CI/CD**: Docker + GitHub Actions

> 상세: [docs/tech-stack.md](docs/tech-stack.md)

### 핵심 데이터 플로우

#### 식사 기록 → 캐릭터 성장 플로우
```
사용자 → 메뉴 결정 (룰렛/월드컵/필터)
       → 메뉴 확정
       → 식사 기록 (메뉴명 검색 + 영양소 자동 매핑)
       → 기록 저장
       → 경험치 획득 (+30 XP 기본 + 보너스)
       → 레벨업 판정
       → 영양소 비율 재계산 (최근 7일)
       → 변신 조건 판정
       → (조건 충족 시) 먹찌 변신 연출 + 도감 등록
```

#### 패널티 플로우
```
Spring Scheduler (매일 실행)
       → 유저별 마지막 기록일 체크
       → 미기록 일수 산정
       → 2일: 먹찌 표정 변경
       → 3일: FCM 푸시 알림 발송
       → 5일: 경험치 감소 플래그 활성화
       → 기록 재개 시 즉시 복구
```

## 🔮 개발 로드맵

### Tier 1 — MVP (출시 목표)
> 룰렛·이상형 월드컵·상황별 필터로 메뉴를 고르고, 텍스트로 식사를 기록하면 먹찌가 성장합니다.

- 메뉴 결정 (룰렛, 이상형 월드컵, 상황별 필터)
- 식사 기록 (텍스트 입력 + 영양소 DB 연동)
- 캐릭터 성장 (경험치, 레벨업, 변신 5종, 패널티)
- 먹부림 캘린더 + 먹찌 도감
- 영양 밸런스 피드백
- 회원가입 / 로그인 (JWT + 소셜)

### Tier 2 — 고도화
> AI 사진 인식, 영수증 OCR, AI 메뉴 추천, 스트릭/퀘스트, 소셜 기능, 캐릭터 커스터마이징

### Tier 3 — 미래 확장
> 그룹 챌린지, 1:1 배틀, 위치 기반 퀘스트, 대화형 AI 상담

> 상세: [docs/feature-tiers.md](docs/feature-tiers.md)
