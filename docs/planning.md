## 메뉴 데이터 초기 구축 전략

식사 기록 및 추천 기능의 전제조건인 메뉴 DB를 구축하는 전략입니다.

### 데이터 소스 및 수집

| 소스 | 수집 방식 | 데이터 범위 | 우선순위 |
|------|----------|-----------|---------|
| 식약처 식품영양성분 DB (MFDS) | API 배치 수집 | 한식 위주 약 5,000건 | 1순위 (MVP) |
| USDA FoodData Central | API 배치 수집 | 양식/글로벌 메뉴 보완 | 2순위 (MVP) |
| 사용자 입력 (미등록 메뉴) | 서비스 운영 중 축적 | 사용자가 직접 입력한 메뉴 | 상시 |

### 초기 세딩 절차

1. **배치 수집 스크립트**: Go CLI 도구로 식약처/USDA API를 호출하여 메뉴 데이터를 `menus` 테이블에 적재
2. **카테고리 매핑**: 수집된 메뉴를 KOREAN, CHINESE, JAPANESE, WESTERN, SNACK, CAFE, OTHER 카테고리로 자동 분류 (키워드 기반 규칙 + 수동 보정)
3. **카테고리별 기본 영양소 산출**: 각 카테고리에 속한 메뉴들의 평균 영양소(칼로리, 탄수화물, 단백질, 지방, 식이섬유, 비타민)를 산출하여 미등록 메뉴의 추정값으로 사용
4. **데이터 품질 검증**: 영양소 값이 비정상적인 항목(칼로리 0 또는 극단값) 필터링

### 카테고리별 기본 영양소 (추정값 기준)

미등록 메뉴 입력 시 사용자가 카테고리를 선택하면 아래 평균값을 자동 적용합니다. 이 값은 초기 세딩 데이터 기반으로 산출되며, 주기적으로 갱신됩니다.

| 카테고리 | 칼로리 (kcal) | 탄수화물 (g) | 단백질 (g) | 지방 (g) |
|---------|-------------|------------|----------|---------|
| KOREAN | 500 | 65 | 20 | 15 |
| CHINESE | 550 | 60 | 18 | 22 |
| JAPANESE | 450 | 55 | 22 | 14 |
| WESTERN | 600 | 50 | 25 | 28 |
| SNACK | 350 | 45 | 8 | 16 |
| CAFE | 300 | 40 | 6 | 14 |
| OTHER | 500 | 55 | 18 | 18 |

> 위 수치는 초기 참고값이며, 실제 세딩 데이터 기반으로 보정합니다.

## 메뉴 결정

### 랜덤 룰렛

사용자가 메뉴를 고르지 못할 때 먹찌가 대신 룰렛을 돌려줍니다.

- 메뉴 풀 구성: 사용자 즐겨찾기 + 최근 기록 메뉴 + 전체 DB 인기 메뉴
- **마스터리 등급이 높은 메뉴**는 룰렛 당첨 확률에 가중치 부여 가능
- 최근 3일 내 먹은 메뉴는 풀에서 제외
- 결과 메뉴를 바로 식사 기록으로 연결 가능

### 상황별 필터

날씨와 기분 태그를 조합해 상황에 맞는 메뉴를 제안합니다.

#### 필터 태그

| 분류 | 태그 |
|------|------|
| 날씨 | 맑음, 흐림, 비/눈, 더움, 추움 |
| 기분 | 기분좋음, 피곤함, 스트레스, 입맛없음, 든든하게 |

- 태그 복수 선택 가능
- 태그 조합별 메뉴 매핑은 초기에는 수동 큐레이션, 이후 사용자 기록 데이터로 보강

### 선호도 기반 메뉴 추천

사용자 데이터를 분석해 개인화된 메뉴 추천 목록을 제공합니다.

#### MVP 추천 (Tier 1)

MVP에서는 구현 가능한 단순 신호만 활용하여 추천합니다.

| 신호 | 데이터 | 활용 방식 |
|------|--------|----------|
| 즐겨찾기 | 사용자 즐겨찾기 목록 | 즐겨찾기 메뉴 우선 노출 |
| 최근 기록 | 최근 식사 기록 메뉴 | 자주 먹는 메뉴 가중치 부여 |
| 명시적 선호 | 메뉴별 좋아요/싫어요 | 싫어요 메뉴 제외, 좋아요 메뉴 가중치 부여 |

- 최근 3일 내 먹은 메뉴는 추천 목록에서 제외 (반복 방지)
- 데이터 부족 시(신규 유저) 카테고리별 인기 메뉴 기반 일반 추천으로 폴백

#### 고도화 추천 (Tier 2)

충분한 사용자 데이터가 축적된 후 4가지 신호를 조합한 정교한 추천으로 확장합니다.

| 신호 | 데이터 | 활용 방식 |
|------|--------|----------|
| 기록 빈도 | 메뉴별 누적 기록 수 | 먹부림 도감 마스터리 등급에 따라 가중치 부여 |
| 시간대 패턴 | 아침/점심/저녁별 메뉴 기록 이력 | 현재 시간대에 자주 먹던 메뉴 우선 |
| 날씨/기분 패턴 | 날씨/기분 태그 + 선택 메뉴 이력 | 현재 날씨/기분과 유사한 상황에서 선택한 메뉴 반영 |
| 명시적 선호 | 메뉴별 좋아요/싫어요 | 싫어요 메뉴 제외, 좋아요 메뉴 가중치 부여 |

- 4가지 신호를 가중합산해 점수 산출 -> 상위 N개 추천

#### 명시적 선호 입력

메뉴 상세 화면 또는 추천 목록에서 좋아요/싫어요를 입력합니다. 입력값은 메뉴별로 저장되며 이후 추천 점수 산출에 즉시 반영됩니다.

#### 미등록 메뉴 처리

- DB에 없는 메뉴 입력 시 메뉴명만 저장하고, 영양소는 카테고리별 기본 영양소 테이블의 평균값으로 추정 적용
- 카테고리별 기본 영양소는 초기 세딩 시 식약처/USDA 데이터에서 카테고리 평균을 사전 산출하여 저장

## 캐릭터 시스템

### 먹찌 생성

- 기본 이름은 "먹찌", 사용자가 원하면 변경 가능
- **생성 완료 직후 '튜토리얼 퀘스트' 자동 시작** (첫 식사 기록 가이드)

### 레벨 & 경험치 시스템

먹찌는 경험치(EXP)를 획득하여 레벨업합니다. 레벨은 진화 단계 전환 및 콘텐츠 해금의 기준이 됩니다.

#### 경험치 지급 기준

| 행동 | 지급 EXP | 비고 |
|------|---------|------|
| 식사 1회 기록 | 10 | 기본 지급 |
| 주간 퀘스트 보상 수령 | 50 | 배경/이펙트/동작 해금 포함 |
| 업적 퀘스트 보상 수령 | 100~500 | 진화/특수 보상에 따라 차등 |
| 친구 태그 수락 | 5 | 태그한 사람, 태그된 사람 양쪽 지급 |
| 패널티(기력 저하) 상태 | 50% 감소 | 기록 재개 시 즉시 복구 |

- 일일 퀘스트는 경험치를 지급하지 않으며, 업적 퀘스트의 누적 조건으로만 작동합니다.

#### 레벨별 필요 경험치

| 레벨 | 해당 레벨 필요 EXP | 누적 EXP | 비고 |
|------|------------------|---------|------|
| 1 | - | 0 | 시작 |
| 2 | 100 | 100 | |
| 3 | 200 | 300 | |
| 4 | 300 | 600 | |
| 5 | 400 | 1,000 | 부화 -> 아기 진화 가능 구간 |
| 10 | 900 | 5,500 | 아기 -> 어린이 진화 가능 구간 |
| 15 | 1,400 | 13,000 | 어린이 -> 청소년 진화 가능 구간 |
| 20 | 1,900 | 23,500 | 청소년 -> 성체 진화 가능 구간 |
| 25 | 2,400 | 37,000 | |
| 30 | 2,900 | 53,500 | 성체 -> 최종 형태 진화 가능 구간 (MAX) |

- 레벨업 공식: 해당 레벨 필요 EXP = 100 x 레벨 (선형 증가)
- 레벨업 시 EXP는 초과분이 이월됨
- 진화 조건은 레벨 + 업적 퀘스트 달성 모두 충족해야 전환

### 퀘스트 시스템

퀘스트 완료 시 먹찌 성장 보상이 지급됩니다. 퀘스트는 일일·주간·업적 3가지로 구성됩니다.

#### 일일 퀘스트 (매일 초기화)

| 퀘스트 | 조건 |
|--------|------|
| 오늘의 식사 | 식사 1회 기록 |
| 삼시세끼 | 하루 3끼 기록 |
| 균형 식단 | 하루 영양 밸런스 달성 (3대 영양소가 각각 권장 비율 범위 이내) |
| 새 메뉴 도전 | 처음 먹는 메뉴 기록 |

일일 퀘스트는 진행 현황만 추적하며 별도 보상 없이 업적 퀘스트의 누적 조건이 됩니다.

#### 주간 퀘스트 (매주 초기화)

| 퀘스트 | 조건 | 보상 |
|--------|------|------|
| 꾸준한 기록 | 7일 연속 기록 | 배경 1종 해금 |
| 균형 식단 주간 | 주간 영양 밸런스 5일 이상 달성 | 이펙트 1종 해금 |
| 다양한 식단 | 서로 다른 메뉴 10종 기록 | 동작 1종 해금 |

#### 업적 퀘스트 (최초 달성 시 보상 지급)

**진화 (외형 변화)**

| 퀘스트 | 조건 | 해금 단계 | 설명 |
|--------|------|----------|------|
| 첫 깨어남 | 첫 식사 기록 | 부화 | 알에서 막 깨어난 아기 먹찌 |
| 일주일 돌봄 | 7일 연속 기록 | 아기 | 작고 귀여운 먹찌 |
| 꾸준한 성장 | 누적 30끼 기록 | 어린이 | 조금 자란 먹찌 |
| 한 달의 식사 | 누적 100끼 기록 | 청소년 | 제법 커진 먹찌 |
| 오랜 동반자 | 누적 200끼 기록 | 성체 | 다 자란 먹찌 |
| 전설의 식사 | 누적 365끼 기록 | 최종 형태 | 위풍당당한 먹찌 |

**동작/모션**

| 퀘스트 | 조건 | 해금 동작 |
|--------|------|----------|
| 시작 | 첫 번째 식사 기록 | 가만히 서기, 눈 깜빡임 |
| 단골 메뉴 | 먹부림 도감 1종 장인 달성 | 고개 끄덕, 기지개 |
| 다양한 식단 | 서로 다른 메뉴 20종 기록 | 점프, 손 흔들기 |
| 사교적인 먹찌 | 친구 먹찌 10회 방문 | 인사, 박수 |
| 댄스 퀘스트 | 30일 연속 기록 | 춤, 회전 |
| 먹방 스타 | 누적 150끼 기록 | 응원, 먹방 포즈 |
| 식단 마스터 | 먹부림 도감 1종 마스터 달성 | 특수 동작 (춤추기) |
| 식단 탐험가 | 서로 다른 메뉴 100종 기록 | 특수 동작 1종 |
| 전설 달성 | 최종 형태 해금 | 전설 동작 1종 |

**배경/환경**

| 퀘스트 | 조건 | 배경 | 설명 |
|--------|------|------|------|
| 시작 | 처음 시작 | 빈 방 | 아무것도 없는 공간 |
| 첫 일주일 | 7일 누적 기록 | 기본 가구 | 작은 테이블, 쿠션 |
| 한 달 기록 | 30일 누적 기록 | 아늑한 방 | 소파, 조명, 러그 |
| 영양 탐험 | 영양 밸런스 20일 달성 | 정원 | 꽃, 나무, 울타리 |
| 꾸준한 돌봄 | 60일 연속 기록 | 넓은 집 | 2층 구조, 발코니 |
| 메뉴 수집가 | 서로 다른 메뉴 50종 기록 | 저택 | 분수, 정원, 조형물 |
| 오랜 친구 | 180일 누적 기록 | 성 | 탑, 성벽, 깃발 |
| 전설의 먹찌 | 최종 형태 해금 | 궁전 | 왕좌, 레드카펫, 보석 장식 |

**이펙트/오라**

### 성장 시스템 (파츠 조합)

먹찌의 외형은 최근 7일 섭취 영양소 비율이 **모든 파츠에 종합적으로 영향**을 줍니다.
레벨업 시점의 영양소 비율이 외형에 반영되며, 달성한 외형은 도감에 기록됩니다.

#### 영양소 입력

최근 7일 평균 섭취 비율 기준 (3대 영양소 + 미량 영양소):
**※ 계산 공식: (메뉴 영양소 함량) x (사용자가 선택한 수량/인분) = 최종 섭취량**

| 영양소 | 기호 | 권장 비율 |
|--------|------|----------|
| 탄수화물 | C | 50~60% |
| 단백질 | P | 15~20% |
| 지방 | F | 20~25% |
| 비타민/무기질 | V | 충분 섭취량 대비 달성률 |
| 식이섬유 | D | 충분 섭취량 대비 달성률 |

#### 파츠 구성 및 결정 규칙

모든 파츠는 전체 영양소 비율의 종합적인 영향을 받습니다.

**체형** (5단계)

| 단계 | 조건 | 외형 |
|------|------|------|
| 매우 마름 | 총 칼로리 권장 대비 50% 미만 | 앙상한 체형 |
| 마름 | 총 칼로리 권장 대비 50~70% | 홀쭉한 체형 |
| 보통 | 총 칼로리 권장 대비 70~110% | 균형 잡힌 체형 |
| 통통 | 총 칼로리 권장 대비 110~140% 또는 탄수화물 65% 이상 | 둥근 체형 |
| 매우 통통 | 총 칼로리 권장 대비 140% 초과 + 탄수화물 65% 이상 | 매우 둥근 체형 |

**근육** (5단계)

| 단계 | 조건 | 외형 |
|------|------|------|
| 없음 | 단백질 10% 미만 또는 총 칼로리 부족 | 부드러운 외곽선 |
| 미약 | 단백질 10~15% | 약간 부드러운 외곽선 |
| 보통 | 단백질 15~22% | 기본 외곽선 |
| 탄탄 | 단백질 22~30% + 총 칼로리 적정 이상 | 탄탄한 외곽선 |
| 뚜렷 | 단백질 30% 이상 + 총 칼로리 적정 이상 | 단단한 외곽선 |

**피부색** (5단계)

| 단계 | 조건 | 외형 |
|------|------|------|
| 창백 | V 달성률 20% 미만 + D 달성률 20% 미만 | 창백한 색감 |
| 연함 | V 또는 D 달성률 20~40% | 연한 색감 |
| 보통 | V 또는 D 달성률 40~60% | 기본 색감 |
| 건강 | V 달성률 60~80% + D 달성률 60% 이상 | 건강한 색감 |
| 싱그러움 | V 달성률 80% 이상 + D 달성률 80% 이상 | 생기 넘치는 색감 |

**표정** (5단계)

| 단계 | 조건 | 외형 |
|------|------|------|
| 지침 | 3대 영양소(C/P/F) 편차 25% 이상 또는 총 칼로리 권장 대비 50% 미만 | 피곤한 표정 |
| 무기력 | 3대 영양소 편차 18~25% 또는 총 칼로리 부족 | 힘없는 표정 |
| 보통 | 3대 영양소 편차 12~18% | 무난한 표정 |
| 기분좋음 | 3대 영양소 편차 6~12% + 총 칼로리 적정 | 미소 짓는 표정 |
| 활력 | 3대 영양소 편차 6% 이내 + 총 칼로리 적정 | 밝고 에너지 넘치는 표정 |

**악세서리** (퀘스트 보상으로 획득)

| 퀘스트 | 조건 | 악세서리 |
|--------|------|---------|
| 첫 돌봄 | 7일 누적 기록 | 기본 악세서리 1종 |
| 식단 탐험 | 서로 다른 메뉴 30종 기록 | 추가 악세서리 1종 |
| 균형 달인 | 영양 밸런스 14일 달성 | 추가 악세서리 1종 |
| 오랜 친구 | 누적 150끼 기록 | 희귀 악세서리 1종 |
| 먹찌 수집가 | 서로 다른 외형 30종 달성 | 희귀 악세서리 1종 |
| 전설 달성 | 최종 형태 해금 | 전설 악세서리 1종 |

#### 외형 조합 예시

| 식습관 | 체형 | 근육 | 피부색 | 표정 | 먹찌 인상 |
|--------|------|------|--------|------|----------|
| 완벽한 밸런스 + 채소 충분 | 보통 | 보통 | 싱그러움 | 활력 | 건강하고 밝은 먹찌 |
| 고단백 + 적정 칼로리 | 보통 | 뚜렷 | 보통 | 기분좋음 | 탄탄하고 활기찬 먹찌 |
| 탄수화물 과다 + 채소 부족 | 통통 | 없음 | 연함 | 무기력 | 힘없는 통통한 먹찌 |
| 극심한 소식 + 편식 | 매우 마름 | 없음 | 창백 | 지침 | 힘없는 먹찌 |
| 고칼로리 + 균형 영양 + 채소 충분 | 통통 | 탄탄 | 건강 | 기분좋음 | 건강한 통통 먹찌 |
| 적정 칼로리 + 약간 편식 | 보통 | 미약 | 연함 | 보통 | 평범한 먹찌 |

- 영양소 비율 조합에 따라 유저마다 고유한 먹찌 외형이 만들어짐 (5x5x5x5 = 625가지)
- 파츠 에셋 수: 5단계 x 4종 = 20개 (악세서리 별도)
- 달성한 외형은 도감에 기록되고, 이전 외형으로 변경 가능 (가역적)
- Rive 애니메이션으로 파츠 조합 + 성장 연출 구현

### 에셋 제작

AI 이미지 생성 도구로 에셋을 제작하고, Rive에서 조합/애니메이션을 구현합니다.

#### 제작 워크플로우

```
1. AI 이미지 생성 (Midjourney / DALL-E / Stable Diffusion)
   - 캐릭터 디자인 확립, 파츠별 변형 생성
   - 화풍 일관성을 위해 동일 프롬프트 스타일 유지
       |
       v
2. 벡터 변환 + 파츠 분리
   - 생성된 이미지를 벡터(SVG)로 변환 (Vectorizer.ai)
   - 체형/근육/피부색/표정 등 레이어별 분리
       |
       v
3. Rive 임포트 + 조합
   - 분리된 파츠를 Rive 에디터에 임포트
   - State Machine으로 영양소 비율 -> 파츠 전환 로직 구성
   - AI Coding Agent (유료)로 동작/상태머신 스크립트 생성 가능
       |
       v
4. Flutter 연동
   - Rive 런타임으로 앱에 적용 (런타임 무료)
```

#### 에셋 제작 우선순위

MVP에서는 대표 에셋만 먼저 제작하고, 나머지는 이후 순차적으로 확장합니다.

**MVP 우선 제작**

| 에셋 | 제작 방법 | 수량 |
|------|----------|------|
| 진화 단계 (대표) | AI 이미지 생성 -> Rive 뼈대 구조 공유 | 부화, 아기, 성체 3단계 |
| 캐릭터 파츠 (대표) | AI 이미지 생성 -> 벡터 변환 -> Rive | 체형 3단계 + 표정 3단계 = 6개 |
| 기본 동작 | Rive에서 직접 애니메이션 | 가만히 서기, 눈 깜빡임 |
| 배경 (대표) | AI 이미지 생성 (정적 일러스트) | 빈 방, 기본 가구 2종 |

### 친구 시스템

#### 친구 검색 및 추가

닉네임 또는 고유 ID로 사용자를 검색해 친구를 추가합니다.

- 검색 탭 진입 시 추천 사용자 목록을 함께 노출
- 친구 요청 -> 상대 수락 시 친구 등록
- 친구 목록에서 프로필 방문 가능

#### 추천 사용자

| 기준 | 설명 |
|------|------|
| 비슷한 식습관 | 최근 기록 메뉴 카테고리가 유사한 사용자 |
| 비슷한 먹찌 | 먹찌 성장 단계가 비슷한 사용자 |
| 인기 사용자 | 연속 기록일이 높은 활발한 사용자 |

추천 목록에는 닉네임, 먹찌 외형, 진화 단계만 노출합니다. 식사 기록 등 상세 데이터는 친구 등록 후 프로필 방문 시에만 공개됩니다.

#### 소셜 인터랙션 (Active Social) 

단순 조회를 넘어 친구와의 유대감을 높이는 기능입니다.

- **응원하기 (Nudge)**: 친구의 먹찌가 '허기짐/배고픔' 상태일 때 응원 알림을 보냅니다. 친구가 기록을 재개하면 두 사람 모두에게 소량의 보상(경험치 등)이 지급됩니다.
- **방명록 (Guestbook)**: 친구 프로필 하단에 짧은 격려의 한 줄을 남길 수 있습니다.

#### 제한

- 최대 친구 수: 100명
- 일일 친구 요청 수: 20회

#### 차단/신고

- 차단: 상대의 검색 결과/추천 목록에서 서로 제외
- 신고: 부적절한 닉네임 등 신고 접수, 관리자 확인 후 처리

#### 친구 태그
- 식사 기록 시 친구 태그
- 친구 목록에서 선택 → 상대방 알림 → 수락 → 양쪽 먹찌 동시에 보너스 경험치 연출

### 패널티 시스템

| 미기록 경과일 | 상태 | 영향 |
|-------------|------|------|
| 0~1일 | 정상 (NORMAL) | 변화 없음 |
| 2일 경과 | 허기짐 (HUNGRY) | 먹찌 표정 변화 (슬픈 표정) |
| 3일 경과 | 배고픔 (STARVING) | 푸시 알림 발송 + 홈 화면 배고픔 연출 |
| 5일 이상 경과 | 기력 저하 (WEAKENED) | 경험치 획득량 50% 감소 (기록 재개 시 즉시 복구) |

- 퀘스트 진행도·외형은 하락하지 않음 (이탈 방지)
- 패널티는 식사 1회 기록 시 즉시 해제

### 알림 전략 (Notification Strategy) 

사용자의 지속적인 방문을 유도하기 위한 맞춤형 푸시 알림 시나리오입니다.

| 상황 | 타이밍 | 메시지 예시 |
|------|--------|------------|
| 식사 권장 | 점심/저녁 시간대 | "오늘의 첫 끼를 기록하고 먹찌를 기쁘게 해주세요!" |
| 패널티 예고 | 3일 미기록 시 | "먹찌가 배고파서 기운이 없어요... 어서 밥을 주세요!" |
| 스트릭 달성 | 목표 1일 전 | "7일 연속 기록 달성까지 단 하루 남았습니다! 힘내세요!" |
| 소셜 반응 | 친구가 응원할 때 | "친구 [닉네임]님이 내 먹찌를 응원했습니다!" |
| 퀘스트 완료 | 퀘스트 달성 시 | "축하합니다! 퀘스트 보상이 도착했어요. 확인해 보세요!" |

- **먹찌의 코멘트**: 마스터 등급 메뉴 클릭 시 먹찌의 전용 리액션 출력

## 기능 단계별 개발 계획

### Tier 1 — MVP (출시 목표)

> 룰렛·상황별 필터로 메뉴를 고르고, 텍스트로 식사를 기록하면 먹찌가 성장합니다.

| 분류 | 기능 | 상세 |
|------|------|------|
| 메뉴 결정 | 랜덤 룰렛 | 먹찌가 대신 룰렛을 돌려주는 인터랙티브 액션 |
| 메뉴 결정 | 상황별 필터 | 날씨·기분 태그 기반 맞춤 추천 |
| 메뉴 결정 | 선호도 기반 메뉴 추천 | 즐겨찾기·최근 기록·명시적 선호 기반 단순 추천 (4가지 신호 분석은 Tier 2) |
| 식사 인증 | 텍스트 직접 입력 | 메뉴명 검색으로 간편 등록, 수량 선택(0.5/1/1.5인분) 반영 |
| 식사 인증 | 음식 영양소 DB 연동 | USDA + 식약처 DB 연동, 검색 실패 시 카테고리 기반 수동 매핑 가이드 제공 |
| 캐릭터 성장 | 파츠 조합 성장 | 영양소 비율에 따른 파츠별 외형 결정 (625가지) |
| 캐릭터 성장 | 퀘스트 진행 연출 | 식사 등록 직후 퀘스트 진행 현황 실시간 반영, 완료 시 보상 지급 (튜토리얼 퀘스트 포함) |
| 캐릭터 성장 | 패널티 시스템 | 3일 미기록 시 먹찌 배고픔 알림 및 상태 노출, 맞춤형 알림 전략(식사 시간대/스트릭 등) 적용 |
| 기록 | 먹부림 캘린더 | 날짜별 먹찌 아이콘 + 식사 기록 |
| 기록 | 먹부림 도감 | 메뉴별 마스터리 등급 및 누적 통계 관리 |
| 기록 | 먹찌 도감 | 달성했던 먹찌 외형 컬렉션 |
| 피드백 | 영양 밸런스 피드백 | 텍스트 기반 다음 메뉴 추천 멘트 |
| 소셜 | 친구 검색 및 추가 | 닉네임/고유 ID 검색 + 추천 사용자 노출, 친구 요청/수락 |
| 소셜 | 인터랙션 확장 | 응원하기(Nudge) 버튼을 통한 리마인드, 프로필 내 방명록(Guestbook) 기능 |

| 소셜 | 차단/신고 | 부적절한 사용자 차단 및 신고 처리 |
| 인증 | 회원가입 / 로그인 | JWT 토큰 발급 및 갱신 |
| 인증 | 소셜 로그인 | Kakao / Google / Apple OAuth |

### Tier 2 — 고도화 (MVP 이후 순차 개발)

> AI가 영양 현황을 분석해 메뉴를 추천하고, 사진·영수증으로 식사를 인증합니다.

| 분류 | 기능 | 상세 |
|------|------|------|
| 메뉴 결정 | 고도화 추천 알고리즘 | 기록 빈도·시간대·날씨기분 패턴·명시적 선호 4가지 신호 가중합산 추천 |
| AI | 음식 사진 자동 인식 | Google Vision API로 음식명 + 영양소 자동 추출 |
| AI | 영수증 OCR | Naver Clova OCR로 메뉴명 자동 추출 |
| AI | AI 메뉴 추천 | 오늘 영양 현황 기반 GPT 추천 + 부족 영양소 룰렛 가중치 |
| AI | 먹찌 일기 | GPT 자동 한 줄 코멘트 |
| 리텐션 | 스트릭 시스템 | 연속 기록 일수 표시 |
| 리텐션 | 식사 패턴 리포트 | 주간/월간 식사 시간 통계 분석 |
| 리텐션 | 스마트 넛지 알림 | 시간·날씨·전날 식단 분석 기반 FCM 푸시 알림 |
| 리텐션 | 시즌 이벤트 | 계절 챌린지, 한정판 캐릭터 외형 |
| 소셜 | 친구 타임라인 피드 | 친구 식사 기록 공유 및 리액션 |
| 소셜 | 먹찌 카드 SNS 공유 | 내 먹찌 카드 이미지 생성 후 공유 |
| 캐릭터 | 인벤토리 시스템 | 악세서리 착용/해제, 아이템별 등급, 코디 저장 |
| 캐릭터 | 외형 커스터마이징 | 추가 파츠, 한정판 악세서리, 획득 아이템으로 먹찌 꾸미기 |
| 캐릭터 | 먹찌 페르소나 | 식습관(육식/채식 등)에 따라 먹찌 성격 부여, 말투/리액션 분화 |
| 피드백 | 선제적 식단 가이드 | 현재 영양 상태 분석 후 다음 식사에서 보충 필요한 식재료 추천 |
